I believe this command contains all problematic files to be directly ignored, run instead of their "colcon build" line. This is due to a high impact on RAM, so instead, install from binaries rather than build from source to spare RAM:sudo apt updatesudo apt install ros-jazzy-ros-gz-sim \\                 ros-jazzy-ros-gz-bridge \\                 ros-jazzy-ros-gz-image \\                 ros-jazzy-gz-ros2-control \\                 ros-jazzy-ros-gz-interfaces \\                 ros-jazzy-ros-gz-sim-demos\# clean memory, in case of bad old installscd ~/workspace/ros2\_kortex\_wsrm -rf build/ install/ log/\# colcon build Kortex packagescolcon build --symlink-install --cmake-args -DCMAKE\_BUILD\_TYPE=Release  --parallel-workers 3NOTE: --parallel-workers 1, this command option limits the amount of parallel processes the command can attempt at once, your computer may be able to handle more or less than 3 at a time (there are around 70 processes to complete).Then, optionally, complete step 6 of the Kortex walkthrough (add workspace to ./bashrc). Otherwise, you will need to run the following each time you use your workspace for ROS2:\# export RMW\_IMPLEMENTATION=rmw\_cyclonedds\_cppsource /opt/ros/jazzy/setup.bashsource ~/workspace/ros2\_kortex\_ws/install/setup.bash**Step 3: Simulate Fake Hardware**From any Ubuntu terminal, after being properly sourced, run:ros2 launch kortex\_bringup gen3.launch.py \\  robot\_ip:=192.168.0.1 \\  # dummy ip / real ip  use\_fake\_hardware:=true \\ # simulate fake hardware  gripper:=robotiq\_2f\_85 \\  fake\_sensor\_commands:=true # simulate fake inputThis will pull up the Kinova Gen3 arm, with a 2f\_85 robotiq gripper at the end.**Step 4: Simulate with MoveIt and Fake Controllers**MoveIt Documentation: 	https://moveit.picknik.ai/main/doc/tutorials/quickstart\_in\_rviz/quickstart\_in\_rviz\_tutorial.htmlAfter proper sourcing, in one terminal, run (Server Client):ros2 launch kinova\_gen3\_7dof\_robotiq\_2f\_85\_moveit\_config robot.launch.py \\  robot\_ip:=192.168.0.1 \\  use\_fake\_hardware:=trueIn another sourced terminal (UI):ros2 launch kinova\_gen3\_7dof\_robotiq\_2f\_85\_moveit\_config moveit\_rviz.launch.pyThis causes RViz to load the robot model and motion planning panel to work with.**Step 5: Create ADL Task Directory in Src**cd ~/workspace/ros2\_kortex\_ws/srcros2 pkg create --build-type ament\_python adl\_tasks \\  --dependencies rclpy moveit\_msgs geometry\_msgs trajectory\_msgsCheck if there is a \_\_init\_\_.py file in adl\_tasks, add if not:cd ~/workspace/ros2\_kortex\_ws/src/adl\_tasksmkdir -p adl\_taskstouch adl\_tasks/\_\_init\_\_.pyBegin writing Python scripts for ADL tasks
